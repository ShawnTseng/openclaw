## 💼 BuddyShopAI - 客戶反饋與架構調整 🆕

### 重大進展
- 🎉 **客戶有回應了！** (打破 168+ 小時沈默)
- 📋 **需求反饋:** 需要 100% 精確模板回答（無 AI 幻覺風險）

### 需求分析
**問題:**
- 目前系統使用 Azure OpenAI 生成動態回答
- 客戶無法保證法律/政策內容的精確措辭
- 對退貨政策、運費說明等關鍵問題需要一字不差

**解決方案:** 混合式回答架構
1. **模板匹配優先:** 關鍵字觸發 → 回傳預設模板（100% 精確）
2. **AI 補充:** 模板未涵蓋 → 使用 AI 生成（保留靈活性）
3. **向下相容:** 現有租戶不受影響

### 技術設計
**架構改動:**
```
User Question
    ↓
Keyword Matching (Template Triggers)
    ↓
    ├─► Match Found → Return Template (exact)
    └─► No Match → AI Generation (current behavior)
```

**Config Schema 擴充:**
```json
{
  "templates": {
    "mode": "hybrid",  // hybrid | template-only | ai-only
    "items": [
      {
        "id": "return_policy",
        "triggers": ["退貨", "退款", "換貨"],
        "response": "【退換貨政策】七天鑑賞期...",
        "responseType": "exact"
      }
    ]
  }
}
```

**新增服務:**
- `TemplateMatchingService.cs` - 關鍵字匹配引擎
- 更新 `LineWebhook.cs` - 混合模式流程

### 實施計畫
- **Week 1 (8-12h):** 核心開發 + 單元測試
- **Week 2 (4-6h):** mrvshop 生產環境測試 + 調優
- **Week 3 (4-6h):** 文件撰寫 + 模板製作指南
- **Total:** 16-24 hours
- **Target:** 2026-03-20

### 效益分析
**Pros:**
- ✅ 100% 保證關鍵回答精確度
- ✅ 降低法律風險（無幻覺）
- ✅ 模板匹配 < 1ms（快 1000x）
- ✅ 省 LLM 成本（模板命中不呼叫 API）

**Metrics:**
