## 🔥 新發現的客戶痛點（P0）

### 當前工作流程：
導入 AI 後，客戶（mrvshop）的工作模式調整為：
- **12:00 中午檢查：** 查看 AI 回覆狀況，處理例外
- **16:00 下午檢查：** 再次查看 AI 狀況，處理例外

### ❌ 現有問題：
- **只有 command line 介面** → 對非技術客戶不友善
- **難以快速掃描狀態** → 無法一眼看出哪些對話需要人工介入
- **處理例外麻煩** → 需要手動切換工具、複製訊息、回覆客戶

### ✅ 需要的解決方案：
**簡易監控 Dashboard** - 讓客戶能輕鬆確認 AI 回覆狀況並快速處理例外

## 🎯 規劃中的功能（優先級排序）

### P0 - 立即解決當前客戶痛點（本月完成）

#### 1. **簡易監控 Dashboard（Web UI）**

**目標場景：**
- 客戶 12:00 和 16:00 打開手機/電腦，5 分鐘內掃完所有狀態
- 快速識別需要人工介入的對話
- 一鍵接手處理例外狀況

**核心功能（MVP）：**

##### 1.1 對話列表（實時監控）
- **顯示：** 最近 N 小時的對話（預設 4 小時，可調整）
- **資訊欄位：**
  - 時間戳記
  - 客戶名稱/ID（如果有）
  - 問題摘要（取前 50 字）
  - AI 回覆狀態：
    - ✅ 正常回覆（綠色）
    - ⚠️ 觸發關鍵字（黃色）- 複雜問題、客戶不滿
    - 🔴 需要人工（紅色）- AI 無法處理、客戶要求真人
    - 🤖 已接手（灰色）- 轉人工處理中
- **排序：** 優先顯示 🔴 需要人工 > ⚠️ 警示 > ✅ 正常
- **篩選：** 全部/僅顯示需要處理

##### 1.2 快速接手功能
- **點擊對話 → 展開詳情：**
  - 完整對話歷史（最近 10 則訊息）
  - AI 回覆內容
  - 客戶情緒標籤（如果有）
- **操作按鈕：**
  - 「接手處理」→ 暫停 AI，通知客戶「專人為您服務」
  - 「標記已讀」→ 確認查看但不接手
  - 「讓 AI 繼續」→ 如果誤判，恢復 AI 回覆
- **快速回覆：**
  - 常用回覆範本（「稍等我幫您確認」、「感謝您的耐心」）
  - 直接輸入自訂回覆（發送到 LINE）

##### 1.3 簡單統計（Dashboard 首頁）
- **今日數據（00:00 - now）：**
  - 總對話數
  - AI 處理數（✅）
  - 人工介入數（🔴）
  - AI 處理率（%）
- **最近 7 天趨勢：**
  - 簡單折線圖（對話數、人工介入率）

##### 1.4 手機友善設計
- **Responsive design**（客戶可能用手機檢查）
- **快速載入**（< 2 秒）
- **推播通知（可選）：** 有 🔴 需要人工時推播到客戶手機

---

**技術方案（最簡架構）：**

```
架構：
- Frontend: Next.js (React) + Tailwind CSS
  - 靜態頁面 + API routes
  - 部署：Vercel（免費額度足夠）
  
- Backend: 現有 Node.js 服務擴展
  - 新增 REST API：
    - GET /api/conversations?hours=4&status=all
    - POST /api/conversations/:id/takeover
    - POST /api/conversations/:id/reply
  
- 資料來源：
  - 現有 LINE webhook logs
  - AI response logs
  - 新增狀態資料庫（SQLite/PostgreSQL）
  
- 認證：
  - 簡單 JWT token 登入
  - 客戶專屬 URL（如 dashboard.buddyshopai.com/mrvshop）
```

**開發時程：**
- Week 1: API 設計 + 資料結構設計 + 對話列表頁面
- Week 2: 快速接手功能 + 統計頁面
- Week 3: 手機優化 + 測試 + 部署
- **Total: 3 週（兼職 15-20h/週）**

**價值：**
- ✅ 立即解決客戶痛點（提升客戶滿意度）
- ✅ 產品差異化（從 CLI 工具 → SaaS 產品）
- ✅ 提價依據（從 $500-1000/月 → $1500-2000/月）
- ✅ 可複制到 SeaFoodAI 和未來客戶

### P1 - 擴展管理功能（下季度）

#### 2. **多 Admin 支援 + 權限管理**

**場景：**
- 客戶有多位員工需要查看 Dashboard
- 不同員工有不同權限（Owner、Admin、Viewer）

**功能：**
- **角色權限：**
  - **Owner:** 全部權限（管理員工、設定 AI、查看所有對話）
  - **Admin:** 處理對話、查看統計（但不能改設定）
  - **Viewer:** 只能查看統計，不能處理對話
- **員工管理：**
  - 新增/移除員工
  - 分配權限
  - 查看操作記錄（誰處理了哪些對話）

**開發時程：** 2-3 週

---

#### 3. **設定多組 AI（不同參數和 system prompt）**

**場景：**
- 客戶有多條產品線（例如：美妝、保健品、服飾）
- 不同產品線需要不同的 AI 回覆風格和知識庫

**功能：**
- **AI 預設集管理：**
  - 建立多組 AI 設定（名稱、system prompt、temperature、FAQ 範本）
  - 根據客戶訊息關鍵字自動切換 AI（例如：提到「面膜」→ 美妝 AI）
- **A/B 測試：**
  - 同時運行兩組 AI 設定，比較哪組效果好（回覆速度、客戶滿意度）

**開發時程：** 3-4 週

---

#### 4. **排班功能**

**場景：**
- 多位 Admin，需要輪班處理例外狀況
- 避免重複接手或漏接

**功能：**
- **排班表：**
  - 設定每日值班人員（例如：12:00-16:00 Alice、16:00-20:00 Bob）
  - 只有值班人員收到推播通知
- **自動分配：**
  - 有新的 🔴 需要人工時，自動分配給當前值班人員

**開發時程：** 2 週

### P2 - 商業模式探索（長期）

#### 5. **是否改變商業模式？用 token 方式變成開源專案？**

**評估：**

##### ❌ Token 經濟不適合 B2B SaaS
- **Token 模式適合：** to C 產品（用戶量大、交易頻繁）
- **B2B 通常：** 訂閱制（月費/年費）- 企業客戶偏好可預測成本

##### ✅ 更適合的開源策略：Freemium
- **開源核心：** 基礎 AI 客服功能（GitHub 公開）
- **閉源進階：** Dashboard、多 Admin、排班、進階統計（付費版）
- **商業模式：**
  - 免費版：自架（需要技術能力）
  - 付費版：託管 SaaS（$500-2000/月）
  - 企業版：客製化（$5000+/月）

##### 🎯 開源的好處：
- **市場教育：** 降低客戶嘗試門檻
- **社群貢獻：** 加速功能開發
- **品牌建立：** 技術權威形象（對澳洲求職有利）
- **Lead Generation：** 免費版用戶 → 付費版轉換

##### ⚠️ 開源的風險：
- **競爭者複製：** 其他公司可能 fork 你的專案
- **維護成本：** 需要處理 issue、PR、文件

**建議時機：**
- **現在：** 專注付費客戶（BuddyShopAI、SeaFoodAI）
- **3-6 個月後：** 有 3-5 個付費客戶，產品穩定後再考慮開源
