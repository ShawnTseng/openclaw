## 驗收流程

1. 本清單涵蓋 **N 個客服情境**，共 **N 種常見問法**
2. 每個情境均有「預期行為」描述，包含訊息數量、關鍵內容、禁止出現的詞
3. **手動驗收**：使用 `customers/{{tenant}}/engineering/qa-scenarios.http` 逐一觸發情境，對照 LINE 實際收到的回覆與本文件的預期行為
4. **自動煙霧測試**：每次部署後由 GitHub Actions 自動執行 `scripts/qa-smoke.sh`，驗證全部情境的 webhook 端點回傳 HTTP 200
5. 如需新增或修改情境，請提供：
   - 客人的問法（至少 2–3 種說法）
   - 預期的標準回覆
   - 是否需要轉接真人客服
