## 💼 BuddyShopAI 下一階段規劃

### 客戶當前痛點（新發現）
**工作流程調整：**
- 導入 AI 後，客戶工作變成：
  - 中午 12:00 檢查一次 AI 回覆狀況
  - 下午 16:00 檢查一次 AI 回覆狀況
- **痛點：** 目前只有 command line，對客戶來說不夠友善
- **需求：** 容易確認 AI 回覆狀況 + 快速處理例外狀況

### 規劃中的功能
- [ ] 優化管理者功能
- [ ] 做後台介面（Web Dashboard）
- [ ] Admin 管理功能
- [ ] 設定多個預設 AI（不同參數和 system prompt）
- [ ] 是否改變商業模式？用 token 方式變成開源專案？

### 🔥 優先級建議（待討論）

#### P0 - 解決當前客戶痛點：
**簡易監控 Dashboard（MVP）**
- **目標：** 讓客戶 12:00 和 16:00 快速檢查 AI 狀態
- **核心功能：**
  1. 最近 N 小時的對話列表（時間、客戶、問題摘要、AI 回覆狀態）
  2. 標記需要人工介入的對話（關鍵字觸發、客戶不滿、複雜問題）
  3. 快速接手功能（一鍵切換 AI → 人工）
  4. 簡單統計（今日對話數、AI 處理率、人工介入率）

- **技術方案：**
  - 最簡單：Web 介面讀取 LINE webhook logs + AI response logs
  - 不需要複雜後台，靜態頁面 + API 即可
  - 手機友善（客戶可能用手機檢查）

#### P1 - 擴展管理功能：
**多 Admin 支援 + 權限管理**
- 不同管理員權限（Owner、Admin、Viewer）
- 設定多組 AI（不同 system prompt 對應不同產品線）
- 排班功能（誰負責哪個時段）

#### P2 - 商業模式探索：
**開源 + Token 經濟？**
- 需要評估：客戶願意為開源版本付費嗎？
- Token 模式適用於 to C，to B 通常還是訂閱制
- 可能路徑：核心開源 + 進階功能收費（Freemium）
