# BuddyShopAI - 下階段戰略規劃

> **文件建立日期：** 2026-03-10  
> **當前版本狀態：** Production（11 天日本旅行零失誤驗證）  
> **客戶：** mrvshop（月費推測 $500-1000）

---

## 🔍 當前痛點分析

### 客戶實際使用場景
**mrvshop 的日常工作流程：**
- **12:00（中午）：** 檢查上午 AI 回覆狀況
- **16:00（下午）：** 檢查下午 AI 回覆狀況
- **需求：**
  1. 快速查看 AI 回覆了哪些訊息
  2. 識別需要人工介入的例外狀況
  3. 接管對話並快速回覆

### 當前解決方案的問題
❌ **只有 command line 介面：**
- 對非技術客戶不友好
- 無法快速瀏覽對話歷史
- 難以過濾和搜尋特定對話
- 無法標記例外狀況
- 缺乏視覺化統計數據

### 核心問題定義
**How might we 讓 mrvshop 用最少時間完成 AI 監控和例外處理？**

---

## 🎯 解決方案優先級

### P0 - 緊急（解決當前痛點）

#### ✅ 後台 Dashboard（必須立即開發）

**核心功能（MVP）：**
1. **對話列表頁**
   - 最近 24 小時對話（可擴展至 7 天）
   - 顯示：時間、客戶名稱、訊息摘要、AI 回覆狀態
   - 快速過濾：全部 / 需要人工 / 已處理 / 已接管

2. **對話詳情頁**
   - 完整對話歷史（像 WhatsApp/LINE 的對話介面）
   - 標示誰回覆的（AI / 人工）
   - 顯示 AI 使用的模板（如果有）
   - 快速接管功能（一鍵切換人工模式）

3. **快速接管模式**
   - 點擊「接管對話」按鈕
   - 直接在 Dashboard 輸入訊息回覆客戶
   - AI 自動暫停該對話的自動回覆
   - 完成後可恢復 AI 模式

4. **異常標記機制**
   - AI 自動標記需要人工介入的情況：
     - 觸發自動轉接關鍵字（已有 7 個）
     - 客戶連續詢問 3 次以上同一問題（AI 可能沒理解）
     - 客戶使用負面情緒詞彙（不滿意、投訴、退款）
   - 管理者手動標記（重要對話、VIP 客戶）

**技術方案：**
- **前端：** Next.js 14 + Tailwind CSS + shadcn/ui
- **後端：** 現有 .NET API 擴展
- **資料庫：** PostgreSQL（已有）
- **即時通訊：** SignalR（.NET 原生，支援即時推送）
- **部署：** Azure App Service（跟現有系統一致）

**開發時間估算：**
- **全職（8h/day）：** 3-5 天
- **兼職（4h/day）：** 1-2 週
- **兼職（2h/day）：** 2-4 週

**ROI 分析：**
- ✅ 直接解決客戶痛點 → 提升產品價值
- ✅ 節省客戶每天 30-60 分鐘（command line → dashboard）
- ✅ 降低客戶流失風險
- ✅ 可作為下一個客戶（SeaFoodAI）的標準功能
- ✅ Demo 素材（澳洲求職作品集）

### P1 - 重要（優化現有流程）

#### ⚠️ 優化管理者功能

**1. 主動通知機制**
- **Email 通知：**
  - 異常對話警報（需要人工介入）
  - 每日報告（12:00, 16:00 自動發送）
  - 週報（每週一 09:00）

- **LINE 通知（可選）：**
  - 即時警報（緊急狀況）
  - 每日摘要（簡化版）

**2. 每日報告內容**
- 今日對話總數
- AI 處理率（自動回覆成功率）
- 人工介入次數
- 需要關注的對話（Top 3）
- 客戶滿意度指標（如果有回饋）

**3. 異常檢測優化**
- **對話過長警報：** 超過 10 輪對話 → 可能 AI 陷入迴圈
- **客戶情緒偵測：** 使用 sentiment analysis API
- **回應時間監控：** AI 回覆超過 30 秒 → 系統可能有問題

**開發時間估算：**
- **Email 通知：** 2-3 天
- **LINE 通知：** 1-2 天
- **異常檢測優化：** 3-5 天
- **總計：** 1-2 週（兼職）

### P2 - 產品化需求

#### 📦 多 AI 配置系統

**1. 預設 AI Persona**
建立不同風格的 AI 助手：

**Persona A - 友善型（預設）：**
```yaml
name: 友善小幫手
temperature: 0.7
system_prompt: |
  你是一個友善、耐心的客服助手...
  使用溫暖的語氣，多用表情符號...
use_case: 一般客戶諮詢、售前問題
```

**Persona B - 專業型：**
```yaml
name: 專業顧問
temperature: 0.3
system_prompt: |
  你是一個專業、精準的顧問...
  使用正式語氣，提供詳細資料...
use_case: B2B 客戶、技術問題
```

**Persona C - 快速型：**
```yaml
name: 快速回應
temperature: 0.1
system_prompt: |
  你專注於快速、簡潔的回答...
  單刀直入，不囉嗦...
use_case: 簡單問題、FAQ
```

**2. 自動切換邏輯**
根據情境自動選擇 AI：
- **客戶類型：** VIP → 友善型，企業 → 專業型
- **問題類型：** FAQ → 快速型，複雜問題 → 專業型
- **時間：** 工作時間 → 專業型，非工作時間 → 友善型

**開發時間估算：** 1-2 週（兼職）

#### 🎨 模板管理 Web UI

**當前問題：**
- 模板存在 JSON 檔案中
- 修改需要編輯程式碼
- 無法即時預覽效果
- 沒有版本控制

**解決方案：**
1. **模板編輯器**
   - 所見即所得編輯（WYSIWYG）
   - 即時預覽
   - 支援變數插入（{客戶名稱}、{產品名稱}）
   - Markdown 支援

2. **版本控制**
   - 模板歷史記錄
   - 一鍵回滾
   - A/B 測試（測試不同模板效果）

3. **模板庫**
   - 預設模板（20+ 常見場景）
   - 自訂模板
   - 模板分類（售前、售後、FAQ、例外處理）

**開發時間估算：** 2-3 週（兼職）

### P3 - 戰略轉型（長期探索）

#### 🪙 Token/開源模式探索

**當前商業模式：**
- 顧問收費（$500-1000/月/客戶）
- 優點：高利潤、深度客製化
- 缺點：難擴展、單一客戶依賴

**Token 模式假設：**
開源核心功能 + 付費進階功能/服務

**可能路徑 A - Freemium SaaS：**
```
Free Tier:
- 基礎 AI 回覆（每月 100 對話）
- 1 個 AI persona
- 7 天對話歷史

Pro Tier ($99/month):
- 無限對話
- 多 AI persona
- 30 天對話歷史
- Email 通知

Enterprise Tier ($499/month):
- 所有 Pro 功能
- 客製化 AI 訓練
- 專屬技術支援
- SLA 保證
```

**可能路徑 B - 開源 + 付費服務：**
```
開源：
- 核心引擎（MIT License）
- 基礎 Dashboard
- 社群支援

付費服務：
- Managed Hosting ($50-200/month)
- 客製化開發（$3000+/project）
- 技術顧問（$150/hour）
- 企業培訓
```

**風險評估：**
❌ **開源風險：**
- 失去程式碼控制權
- 競爭對手直接抄襲
- 社群管理成本高
- 免費用戶支援負擔

✅ **開源優勢：**
- 快速擴散和採用
- 社群貢獻新功能
- 品牌知名度提升
- 吸引投資人注意

**建議：**
🚦 **現階段不建議開源**
- 目前只有 1 個付費客戶（mrvshop）
- 產品還在驗證 product-market fit
- 沒有護城河（技術不複雜，容易被複製）
- 時間緊迫（87 天後去澳洲）

**何時考慮開源/Token 模式？**
- ✅ 有 5+ 個付費客戶（證明市場需求）
- ✅ 月營收 $5000+（證明商業模式成立）
- ✅ 建立技術護城河（專有數據、產業知識庫）
- ✅ 有團隊支援（不只你一人）
